记录下git常用的方法

新开的项目

  1. $ mkdir git

  2. $ cd git

  3. $ git init

  4. $ touch README.md

  5. $ git add README.md

  6. $ git commit -m “log message”

  7. $ git remote add origin [name].git

  8. $ git push -u origin master

已有项目

  1. $ cd git

  2. $ git remote add origin [name].git

  3. $ git push -u origin master

  4. $ touch README.md

  5. $ git add README.md

  6. $ git commit -m “log message”

  7. $ git remote add origin [name.git]

  8. $ git push -u origin master

修改远程已有项目

  1. $ git clone [name].git

  2. $ cd git

  3. $ git branch

  4. $ git branch yourBranch

  5. $ git checkout yourBranch

  6. coding …

  7. $ git commit

  8. $ git add -A(or your file’s name)

  9. $ git commit -m your message or git commit

  10. $ git push

本地merge

  1. $ git checkout master

  2. $ git merge yourBranch

  3. $ git push

其他

  • $ git pull 更新本地代码
  • $ git remote rm [name].git 删除remote
  • $ git remote set-url —push [name] [newUrl] 更新remote

误入歧途

  1. [modified content] 当拷贝整个文件夹时，提交时出现 [modified content]时，文件夹无法add到commit资源中，这是因为拷贝整个文件夹时，把.git文件都拷过去了，这样就会出现原有项目有.git，而我们拷贝的文件夹里也有.git导致传不上去，所以建议 不要直接拷贝整个文件夹(含.git)，而是新建一个文件夹，然后再把内容拷贝过去。->相关文章
