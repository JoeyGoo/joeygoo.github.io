发布文章

  1. 在 source > _posts 下创建新的 [name].md

  2. 写..

  3. $ hexo generate 静态化处理

  4. $ hexo deploy 发布到github

Empty reply from server

当使用hexo deploy时，实际调用的是git指令去push到github(或其他仓库上)，今天执行hexo deploy时出现了标题中的错误,如下

fatal: unable to access ‘https://github.com/JoeyGoo/joeygoo.github.io.git‘: Empty reply from server;

后来查明原因，发现是github上HTTPS通道被墙的原因，访问失败，服务器无反应，解决方式如下：

  1. 改用SSH通道，点击查看生成SSH keys

  2. 将github上的项目选择SSH通道,并copy该地址

  3. 在hexo项目中，改变通道(HTTPS/SSH)方式需在项目的根目录的_config.yml文件中 改变deplay下的repo地址(对应github上的项目地址)
